# 遗传算法数学求解器开发规划

## 1. 项目概述

### 1.1 项目名称

建议项目名称：

- MathGA Solver
- 遗传算法数学求解器
- 基于遗传算法的高中数学数值求解平台

### 1.2 项目定位

本项目计划开发一个基于遗传算法的网页端数学求解程序，主要面向高中及以下数学问题。

项目不是传统意义上的“精确符号计算器”，而是一个“数值型智能搜索求解器”。它通过遗传算法不断进化候选答案，寻找函数最值、方程近似解、函数拟合参数、几何最优解等。

推荐定位：

> 一个基于遗传算法的高中数学数值求解与建模演示平台。

### 1.3 核心思想

不同数学问题都可以转化为：

> 搜索某些变量，使目标函数达到最优。

例如：

| 数学问题 | 遗传算法搜索对象 | 优化目标 |
|---|---|---|
| 求最大值 | x | f(x) 最大 |
| 求最小值 | x | f(x) 最小 |
| 解方程 | x | \\|f(x)\\| 最小 |
| 函数拟合 | 参数 a, b, c | 误差最小 |
| 几何最值 | 几何变量 | 面积、距离等最优 |
| 已知最值反推函数 | 函数参数 | 目标最值误差最小 |

因此，项目底层最重要的是：

> 通用遗传算法优化器 + 不同问题的适应度函数。

## 2. 项目适用范围

### 2.1 适合用遗传算法解决的问题

遗传算法适合解决以下类型的问题：

1. 函数最大值和最小值
2. 方程近似求解
3. 多解搜索
4. 函数拟合
5. 已知条件反推函数参数
6. 多变量函数优化
7. 几何最值问题
8. 不等式近似求解
9. 参数方程搜索
10. 实际建模类优化问题

### 2.2 不适合用遗传算法作为核心的问题

以下问题不适合作为项目核心功能：

1. 精确因式分解
2. 精确代数化简
3. 严格数学证明
4. 自动推导解析式
5. 自动求根式精确解
6. 完全自动识别所有高中数学题型

例如：

```text
x^2 - 2 = 0
```

遗传算法可以得到：

```text
x ≈ 1.41421356
```

但不能自然推出：

```text
x = √2
```

除非后期额外加入“近似值转根式”的模块。

```ts
import { compile } from "mathjs";
```

## 4. 用户交互设计

### 4.1 总体原则

为了最大限度减少用户输入错误，输入方式应优先采用：

> 题型选择 + 模板选择 + 下拉框 + 数字输入框 + 快捷按钮

不建议让用户直接手写完整区间、完整函数或复杂数学符号。

推荐原则：

```text
能选择就不手写；
能用模板就不拼表达式；
必须输入数字时才显示数字输入框；
自由表达式输入作为高级模式，而不是默认模式。
```

第一版可以采用“模板选择优先”的方式：

```text
选择函数类型：
- 一次函数：ax + b
- 二次函数：ax² + bx + c
- 三角函数：a·sin(bx + c) + d
- 指数函数：a·bˣ + c
- 自定义表达式：高级模式
```

用户选择模板后，只需要填写或选择参数：

```text
二次函数
f(x) = ax² + bx + c

a：1
b：-4
c：3
自变量：x
```

程序内部再自动生成表达式：

```text
x^2 - 4*x + 3
```

界面显示为：

```text
x² - 4x + 3
```

这样用户不需要记忆 `*`、`^`、括号、函数名等语法。

### 4.2 数学符号显示设计

界面上展示给用户看的数学符号，应尽量使用接近课本和试卷的写法。

推荐采用两层表示：

```text
用户看到的显示表达式：ax² + bx + c
程序内部计算表达式：a*x^2 + b*x + c
```

也就是说：

```text
界面负责易读；
内部负责稳定计算。
```

常见符号映射建议：

| 用户看到 | 内部表达式 | 说明 |
|---|---|---|
| x² | x^2 | 平方 |
| x³ | x^3 | 立方 |
| a·x 或 ax | a*x | 乘法 |
| √x | sqrt(x) | 平方根 |
| \|x\| | abs(x) | 绝对值 |
| π | pi | 圆周率 |
| e | e | 自然常数 |
| sin x | sin(x) | 正弦 |
| cos x | cos(x) | 余弦 |
| tan x | tan(x) | 正切 |
| x ≥ 0 | x >= 0 | 大于等于 |
| x ≤ 0 | x <= 0 | 小于等于 |

例如用户选择二次函数时，页面显示：

```text
f(x) = ax² + bx + c
```

用户填写参数后，页面预览显示：

```text
f(x) = x² - 4x + 3
```

程序内部实际使用：

```text
1*x^2 + (-4)*x + 3
```

高级模式也应尽量提供“友好按钮”，按钮文字使用数学符号：

```text
x²
x³
√
|x|
π
sin
cos
tan
≥
≤
```

按钮点击后，插入到内部表达式输入框的内容可以是：

```text
^2
^3
sqrt()
abs()
pi
sin()
cos()
tan()
>=
<=
```

这样用户看到的是熟悉的数学符号，程序仍然使用 mathjs 容易解析的表达式。

### 4.3 区间输入设计

不要让用户手写：

```text
[-2π, 2π)
```

推荐拆成：

```text
左端点符号：[ 或 (
左端点数值：-2π 或 -6.28
右端点数值：2π 或 6.28
右端点符号：] 或 )
```

页面效果：

```text
区间： [  -2π  ,  2π  ]
```

其中 `[`, `(`, `]`, `)` 使用下拉框选择。

对应数据结构：

```ts
type Interval = {
  left: number;
  right: number;
  includeLeft: boolean;
  includeRight: boolean;
};
```

### 4.6 函数模板选择设计

第一版推荐把函数输入改为模板选择。



这样可以显著降低用户输错表达式的概率。

### 4.8 表达式输入设计

默认情况下，不建议直接显示自由表达式输入框。

推荐分成两种模式：

```text
普通模式：选择函数模板并填写参数
```

普通模式适合大多数高中及以下数学场景。

高级模式用于模板无法覆盖的函数，例如：

```text

|x - 1| / (x + 2)
```

高级模式中仍然保留辅助输入：

```text
函数表达式输入框
快捷符号按钮
错误提示
示例按钮
```

常用快捷按钮：

```text

+
-
×
/
x²
x³
(
)
sin
cos
tan
√
|x|
π
e
```

按钮点击后，将对应内容插入到表达式输入框光标位置。

高级模式需要明确提示：

```text
高级模式需要按程序语法输入，例如 2*x、x^2、sqrt(x)。
```

但页面预览仍应尽量转换成用户容易看懂的样子，例如：

```text
输入：2*x + sqrt(x)
预览：2x + √x
```

### 4.9 输入方式优先级

整体输入方式建议按以下优先级设计：

```text
1. 题型模板
2. 函数模板
3. 参数数字输入
4. 下拉框选择变量、目标、区间端点类型
5. 快捷按钮拼接表达式
6. 高级自由表达式输入
```

第一版可以只实现：

```text
函数最值：
- 选择函数模板
- 选择自变量
- 填写参数
- 输入区间
- 选择最大值、最小值或两者

方程求解：
- 选择左侧函数模板
- 选择右侧常数或函数模板
- 输入区间
- 选择找一个解
```

自定义表达式可以作为“高级模式”保留，但不作为主要入口。

## 8. 遗传算法核心设计

### 8.1 通用流程

遗传算法统一流程：

```text
1. 初始化种群
2. 计算每个个体的适应度
3. 排序并保留精英个体
4. 选择父代
5. 交叉生成子代
6. 对子代进行变异
7. 形成新一代
8. 判断是否提前停止
9. 输出最优个体
```

### 8.2 默认参数建议

```text
种群数量：80
最大迭代次数：300
交叉概率：0.8
变异概率：0.1
精英保留数量：2
提前停止误差：1e-7
连续无提升代数：50
```

第一版可以使用：

```text
种群数量：60
最大迭代次数：200
交叉概率：0.8
变异概率：0.1
精英保留数量：2
```

### 8.3 实数编码

本项目推荐使用实数编码，而不是二进制编码。

一元函数个体：

```ts
{
  x: 3.14
}
```

多变量函数个体：

```ts
{
  x: 1.2,
  y: -0.5
}
```

函数拟合个体：

```ts
{
  a: 2.01,
  b: -0.03,
  c: 1.00
}
```

### 8.4 选择策略

第一版建议使用锦标赛选择。

思路：

```text
随机选出若干个个体
从中选择适应度最高的个体作为父代
```

优点：

- 实现简单
- 不要求适应度必须为正数
- 适合函数最大值和最小值问题

### 8.5 交叉策略

实数编码可以使用算术交叉。

例如父代：

```ts
parent1 = { x: 2 }
parent2 = { x: 8 }
```

随机系数：

```ts
alpha = 0.3
```

子代：

```ts
child.x = alpha * parent1.x + (1 - alpha) * parent2.x
```

### 8.6 变异策略

实数变异可以使用随机扰动：

```ts
x = x + randomNoise
```

变异后必须限制在区间范围内。

例如：

```ts
x = Math.max(min, Math.min(max, x));
```

对于开区间，需要避免等于端点。

## 9. 各功能模块设计

## 9.1 函数最值模块

### 功能目标

用户选择函数模板、填写参数和区间，程序求最大值、最小值或同时求两者。

### 输入示例

```text
函数类型：二次函数
参数：a=1, b=-4, c=3
预览：f(x) = x² - 4x + 3
区间：[0, 5]
目标：最大值和最小值
```

### 求最大值

适应度函数：

```ts
fitness = f(x)
```

### 求最小值

可以转化为最大化：

```ts
fitness = -f(x)
```

### 注意事项

如果函数值为非法值：

```ts
NaN
Infinity
-Infinity
```

则给该个体很差的适应度。

### 边界处理

闭区间 `[a, b]`：

```text
端点可能是最大值或最小值
需要单独计算 f(a), f(b)
```

开区间 `(a, b)`：

```text
端点不能取
如果结果非常接近端点，需要提示可能只是上确界或下确界
```

例如：

```text
f(x)=x
区间：(0,1)
```

程序可能得到：

```text
x ≈ 0.999999
f(x) ≈ 0.999999
```

但真正最大值不存在，只存在上确界 1。

页面应提示：

```text
结果接近未包含端点，可能只是上确界或下确界，不一定是真正最值。
```

## 9.2 方程求解模块

### 功能目标

用户通过模板和符号选择组成方程，程序求近似解。

### 输入示例

```text
x² - 4 = 0
区间：[-10, 10]
```

### 转化方式

将方程：

```text
left(x) = right(x)
```

转化为：

```text
left(x) - right(x) = 0
```

然后最小化误差：

```ts
error = Math.abs(left(x) - right(x));
fitness = 1 / (1 + error);
```

### 单解搜索

一次遗传算法通常容易找到一个解。

输出：

```text
x ≈ 2.000001
误差 ≈ 0.0000003
```

### 多解搜索

如果要找多个解，可以采用分区间搜索：

```text
1. 将大区间分成多个小区间
2. 每个小区间单独运行遗传算法
3. 收集结果
4. 合并非常接近的解
```

例如：

```text
[-10, 10] 分成 40 个小区间
每段长度 0.5
```

## 9.4 已知最值反推函数参数模块

### 功能目标

用户给出函数模板、区间、目标最大值和目标最小值，程序搜索一组可能的参数。

注意：只知道最大值和最小值通常不能唯一确定函数，所以本功能应称为：

> 根据条件拟合一个可能的函数。

### 输入示例

```text
函数模板：ax² + bx + c
区间：[0, 2]
目标最大值：5
目标最小值：1
```

### 搜索对象

```ts
{
  a: number,
  b: number,
  c: number
}
```

### 适应度设计

对每一组参数，先生成函数，然后计算该函数在区间内的近似最大值和最小值。

误差：

```ts
error =
  Math.abs(actualMax - targetMax) +
  Math.abs(actualMin - targetMin);

fitness = 1 / (1 + error);
```

### 实现建议

为了降低复杂度，内层最值计算第一版可以使用密集采样，而不是再嵌套一次遗传算法。

## 12. Next.js 实现注意事项

### 12.1 客户端组件

遗传算法在浏览器中运行，涉及 useState 和用户交互的组件需要加：

```tsx
"use client";
```

例如：

```tsx
"use client";

import { useState } from "react";
```

### 12.2 不要直接使用 eval

不要直接使用：

```ts
eval(expression)
```

原因：

```text
1. 有安全风险
2. 用户输入难以控制
3. 错误处理困难
```

建议使用 mathjs 的 compile 和 evaluate。

### 12.3 表达式异常处理

用户可能输入非法表达式，例如：

```text
x^^2
sqrt(-1)
1 / 0
```

计算时需要判断：

```ts
if (!Number.isFinite(value)) {
  return null;
}
```

### 12.4 开区间提示

对于开区间，如果结果非常接近未包含的端点，需要提示：

```text
该结果接近未包含端点，可能只是上确界或下确界，不一定是真正最大值或最小值。
```

### 12.5 结果为近似值

页面必须明确说明：

```text
本程序使用遗传算法进行数值搜索，结果为近似值，不保证是严格解析解。
```

## 18. 重要提醒

1. 遗传算法结果是近似值，不是严格解析解。
2. 对于开区间问题，最大值或最小值可能不存在。
3. 对于方程问题，一次运行不一定找到所有解。
4. 对于函数反推问题，结果通常不唯一。
5. 第一版要控制范围，先完成函数最值和方程求解。
6. 不要一开始追求“万能数学求解器”。
7. 项目重点应放在遗传算法思想、数值搜索过程和网页交互展示上。
